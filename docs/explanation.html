Git Commands Reference

A terminal-style quick reference for Git workflows and internals.

Repository Setup

git init

Initialize a new Git repository in the current directory. Creates a hidden .git folder that tracks all version history and configuration for your project.

git clone <url>

Clone an existing repository from a remote source. Downloads the entire project history and sets up a local working copy linked to the remote.

Basic Workflow

git status

Show the current state of the working directory and staging area. Displays modified, staged, and untracked files.

git add <file>

Stage changes for the next commit by moving them into the staging area.

git commit -m "message"

Record a snapshot of staged changes with a descriptive message.

git restore <file>

Discard uncommitted changes and revert file to last committed state.

git rm <file>

Remove a file and stage its deletion for the next commit.

Branching & Merging

git branch

List, create, or delete branches for isolated development.

git switch <branch>

Switch between branches using a modern, focused command.

git merge <branch>

Merge another branch into the current branch, combining histories.

git rebase <branch>

Reapply commits onto a new base to create a linear history.

Remote Repositories

git remote -v

Show configured remote repositories and their URLs.

git fetch

Download changes from a remote without modifying working files.

git pull

Fetch and merge remote changes into the current branch.

git push

Upload local commits to a remote repository.

History & Inspection

git log

Display commit history in chronological order.

git diff

Show line-by-line differences between commits or working changes.

git show <commit>

Inspect a specific commit and its changes.

Undoing Changes

git reset <commit>

Move branch pointer to a specific commit (can affect staging and working directory).

git revert <commit>

Create a new commit that safely undoes a previous commit.

Stashing

git stash

Temporarily save uncommitted changes for later use.

git stash pop

Reapply stashed changes and remove them from stash.

Tagging

git tag

Mark specific commits for releases or versioning.

Plumbing Commands

git cat-file -p <object>

Inspect Git objects (blobs, trees, commits) at a low level.

git hash-object <file>

Generate SHA hash for a file and optionally store it in Git’s object database.

git ls-tree <tree>

List contents of a tree object from a commit.

git rev-parse <ref>

Resolve branch names or tags into commit hashes.

Notes

  • Porcelain commands are user-friendly (commit, push, pull).
  • Plumbing commands are low-level Git internals for advanced usage.
